The system under test is a 5G Standalone (SA) core network where user traffic generated by UEs is forwarded through the User Plane Function (UPF) toward external data networks. 
User-plane traffic is carried over the N3 interface using GTP-U between the gNB and the UPF, and forwarded on the N6 interface as plain IP traffic. 
Monitoring is performed at the 5G core, focusing on user-plane traffic patterns and UPF resource usage.

Protocols and interfaces:
- Interfaces: N3 (gNB <-> UPF), N6 (UPF <-> Data Network)
- Protocols: GTP-U (N3), IP (N6), PFCP (N4, optional if session state changes)

Property: The UPF must sustain normal user-plane forwarding performance and detect abnormal traffic patterns originating from UE(s) that may cause data-plane saturation or resource exhaustion, impacting service quality for other users.

Trigger: One or more UEs generate excessive user-plane traffic after successful PDU session establishment, leading to sustained high-rate GTP-U flows toward the UPF and external data networks.

Expected traffic pattern (baseline):
Under normal conditions:
- GTP-U traffic on N3 reflects moderate, session-bound throughput per UE
- Traffic on N6 remains within expected bandwidth and latency bounds
- UPF forwarding state remains stable over time
- In the anomalous case, traffic volume and packet rates increase sharply and persist over time.

Observable Evidence
The monitoring system must inspect:
- GTP-U (N3): packet rate, byte volume, TEID distribution, UE/session correlation
- IP (N6): aggregate throughput, packet loss indicators, latency variation
- Temporal patterns: sustained high traffic rates from specific UE(s) or sessions
- Optional PFCP: session modification or teardown events if overload causes session flapping

Detection Condition
An alert must be raised when:
- user-plane traffic exceeds defined throughput or packet-rate thresholds over a time window, or
- persistent traffic saturation correlates with increased packet loss or latency at the UPF.

Response:
Upon detection, the system must generate an alert summarizing affected interfaces (N3/N6), traffic metrics (rate, volume, duration), involved UE or session identifiers, and the observation window, enabling correlation with UPF performance degradation.
